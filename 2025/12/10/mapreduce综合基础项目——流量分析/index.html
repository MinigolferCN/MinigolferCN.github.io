MapReduce综合基础项目——流量分析

在学习了基本的MapReduce项目构建和业务实现方法后，我们来完成一个有关流量分析的基础综合项目。

一、项目描述

这个项目基于data_flow流量记录文件，要求对其进行数据处理。

1.数据总览

data_flow文件的数据格式如下：

以下是用Markdown语法呈现的表格：

序号 字段 字段类型 描述
0 reportTime long 记录报告时间戳
1 misstidn String 记录号码
2 apmac String AP mac
3 acmac String AC mac
4 host String 访问的网址
5 siteType String 网站种类
6 upPackNum long 上行数据包数，单位：个
7 downPackNum long 下行数据包数，单位：个
8 upLoad long 上行总流量。要注意单位的转换，单位：byte
9 downLoad long 下行总流量。要注意单位的转换，单位：byte
10 httpStatus String HTTP Response 的状态

2.项目需求

本项目分为三个子需求：将原数据中同一电话号码的数据的upPackNum、downPackNum、upLoad和downLoad数据求和；将需求一的结果中的结果按照upPackNum降序排序；将需求一的结果按照手机号开头三位分成“135”、“136”、“137”和其他四类

二、需求实现

1.流量求和

求和任务本质上延续了wordcount任务的逻辑。不过由于要求和的是四列数据，要定义新的java类用于数据存放和传输。

由于在这个项目中不用进行排序，所以只需要实现一个Writable接口来实现序列化。

①.FLowBean

②.FlowCountMapper

K1-V1是默认的<偏移量，行数据>，要将其转化为K2-V2即 <手机号，< UpFlow, DownFlow, UpCountFlow, DownCountFlow > >。通过将行数据按制表符分割，提取各列数据来组成一个flowbean类，再与手机号一起写入上下文中。

③.FlowCountReducer

经过Shuffle阶段，V2变成了可迭代的变量，包含了同一个电话号码的多个V2。对其进行迭代求和作为V3，与K2组成新的K3-V3输入上下文中。

④.FlowCountDriver

⑤.运行结果

2.流量排序

这个任务要求对任务一的结果进行排序，依照upFlow进行降序排序。

①.FlowBean

由于要实现排序，所以要实现一个WritableComparable。

②.FlowSortMapper

Mapper逻辑与任务一的相同。

③.FlowSortReducer

由于不需要求和，K2-V2与K3-V3相同，Reducer仅仅将K2-V2作为K3-V3传到上下文中。

④.FlowSortDriver

3.手机号分区

分区任务要创建一个Partitioner的子类。这里我没有采用自定义的类来存放数据，而是把行数据的解析工作放在了Partitioner类中完成。

①.FlowPartitionMapper

由于我没有是指Bean类，因此Mapper阶段不用进行操作，仅将数据输入上下文，交给Partition阶段进行解析和排序。

②.FlowPartitioner

对于传入的数据用制表符进行分割，提取第一个数据即为手机号，根据手机号开头来返回不同的分区序号。

③.FLowPartitionReducer

Reducer阶段也不需要完成额外的工作。

④.FlowPartitionDriver

Driver中要指定Partition的数量，即job.setNumReduceTasks(4)。

⑤.运行结果

三、总结

完成了这个综合项目代表着我已经掌握了MapReduce的基础编程思想，为之后的学习和理解课程实验打下了基础。
